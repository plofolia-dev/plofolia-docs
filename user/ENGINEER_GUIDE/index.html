    Plofolia — エンジニア向け技術概要¶

    クローズドベータに参加いただいたエンジニアの方へ。
    Plofolia の設計思想・技術スタック・API の概要をまとめた資料です。

    1. プラットフォーム概要¶

    Plofolia（プロフォリア） は、クリエイター向けの SNS + ポートフォリオプラットフォームです。

    コンセプト¶

    • Plot（WIP の投稿） + Portfolio（完成作品の整理） を一か所に
    • X でシャドウバンを受けたイラストレーター・映像作家・AI クリエイターの受け皿
    • NSFW・AI 生成コンテンツを差別しない透明なアルゴリズム

    独自機能（他プラットフォームにない点）¶

    機能 概要
    タイムコメント 動画・音声投稿に時刻付きコメント。YouTube Live 風のオーバーレイ表示
    タイムライン分離 手描き / AI 生成 / 映像 の3タブ。見たいカテゴリだけ選べる
    シャドウバン可視化 クリエイター分析ダッシュボードで自分へのアルゴリズム影響を確認
    AI 学習拒否宣言 ワンクリックで宣言。プロフィールにバッジ表示、robots_directives に反映
    地域フィルタリング 設定した国籍のクリエイターの投稿のみタイムラインに表示

    2. システムアーキテクチャ¶

    サービス構成¶

    すべて単一の EC2 インスタンス（t3.micro / ap-northeast-1）上で稼働しています。

    技術スタック¶

    層 技術・ライブラリ
    フロントエンド Next.js 14 App Router + TypeScript + Tailwind CSS
    状態管理 Zustand（persist で localStorage 永続化）+ TanStack Query
    多言語 next-intl（ja / en / zh / ko / es の5言語）
    バックエンド Express.js + TypeScript + Drizzle ORM
    DB PostgreSQL 16
    認証 JWT（アクセス 15 分 + リフレッシュ 7 日）+ bcrypt
    OAuth Passport.js + passport-oauth1（Google OAuth 2.0 / X OAuth 1.0a）
    バリデーション Zod（API リクエスト）
    メール AWS SES（nodemailer 経由）
    PWA Service Worker（キャッシュファースト + ネットワークファースト）
    CI GitHub Actions（PostgreSQL サービスコンテナ + Jest）

    3. 認証フロー¶

    メール + パスワード登録¶

    sequenceDiagram
    participant C as クライアント
    participant API as sol API
    participant DB as PostgreSQL
    participant Mail as メール (SES)

    Note over C,Mail: 新規登録フロー
    C->>API: POST /auth/register
    API->>API: DNS MX レコード検証
    API->>DB: ユーザー作成 (emailVerifiedAt = null)
    API->>Mail: 確認メール送信（24h 有効トークン）
    API-->>C: 201 Created（まだログイン不可）

    C->>API: GET /auth/verify-email?token=XXX
    API->>DB: emailVerifiedAt = NOW()
    API-->>C: 200 認証完了

    Note over C,Mail: ログインフロー
    C->>API: POST /auth/login
    API->>DB: ユーザー検索・パスワード照合・失敗回数チェック
    API->>DB: refresh_tokens に新トークンを保存
    API-->>C: accessToken (body) + refreshToken (httpOnly Cookie)

    Note over C,API: 15 分後、アクセストークン期限切れ
    C->>API: POST /auth/refresh（Cookie 自動送信）
    API->>DB: 旧トークンを used_at でマーク（盗難検知用に保持）
    API->>DB: 新トークンを保存
    API-->>C: 新 accessToken + 新 refreshToken Cookie

    JWT ペイロード¶

    {
  "userId": "uuid",
  "username": "string",
  "role": "user | admin | moderator",
  "isAdult": true,
  "preferredLocale": "ja"
}

    リフレッシュトークンローテーション¶

    • 使用するたびに新しいトークンを発行、旧トークンを used_at でマーク
    • 使用済みトークンの再使用 → 盗難と判断し全セッションを即時無効化

    OAuth（Google / X）¶

    • Google: OAuth 2.0（passport-google-oauth20）
    • X: OAuth 1.0a（passport-oauth1 で独自実装。passport-twitter は xmldom CVE のため廃止）
    • 既存メールアドレスと OAuth を後から連携可能

    4. 主要 API エンドポイント¶

    ベース URL: https://plofolia.com/api/v1

    認証が必要なエンドポイントは Authorization: Bearer <accessToken> ヘッダーを付与。

    認証¶

    メソッド パス 認証 概要
    POST /auth/register 不要 新規登録（メール確認が必要）
    POST /auth/login 不要 ログイン（メールまたはユーザー名）
    POST /auth/refresh 不要 アクセストークン更新
    POST /auth/logout 不要 ログアウト
    GET /auth/verify-email?token= 不要 メールアドレス確認
    GET /auth/check-verified?email= 不要 認証済みか確認
    POST /auth/forgot-password 不要 パスワードリセットメール送信
    POST /auth/reset-password 不要 パスワードリセット実行
    POST /auth/resend-verification 不要 確認メール再送

    ユーザー¶

    メソッド パス 認証 概要
    GET /users/:username 不要 プロフィール取得
    PATCH /users/me 必要 プロフィール更新
    GET /users/me/browse-regions 必要 閲覧地域フィルター取得
    PUT /users/me/browse-regions 必要 閲覧地域フィルター更新
    GET /users/me/protection 必要 AI 学習拒否宣言の取得
    PUT /users/me/protection 必要 AI 学習拒否宣言の更新
    POST /users/:username/follow 必要 フォロー
    DELETE /users/:username/follow 必要 フォロー解除
    POST /users/:username/block 必要 ブロック
    POST /users/:username/mute 必要 ミュート

    投稿¶

    メソッド パス 認証 概要
    POST /posts 必要 投稿作成（multipart/form-data）
    GET /posts/:id 不要 投稿詳細取得
    DELETE /posts/:id 必要 投稿削除（自分の投稿のみ）
    POST /posts/:id/like 必要 いいね
    DELETE /posts/:id/like 必要 いいね解除
    POST /posts/:id/repost 必要 リポスト
    POST /posts/:id/bookmark 必要 ブックマーク
    POST /posts/:id/report 必要 投稿を報告
    GET /posts/bookmarks 必要 ブックマーク一覧

    タイムライン¶

    メソッド パス 認証 概要
    GET /timeline/global 不要 グローバルタイムライン（全員分）
    GET /timeline/following 必要 フォロー中タイムライン

    クエリパラメータ: ?cursor=<id>&limit=20&type=drawing|ai|video

    タイムコメント¶

    メソッド パス 認証 概要
    GET /posts/:id/timed-comments 不要 タイムコメント一覧取得
    POST /posts/:id/timed-comments 必要 タイムコメント投稿
    DELETE /timed-comments/:id 必要 タイムコメント削除（自分のみ）

    POST ボディ: { "timecode": 12500, "body": "コメント本文" } （timecode は ms 単位）

    メッセージ（DM）¶

    メソッド パス 認証 概要
    GET /messages/threads 必要 スレッド一覧
    POST /messages/threads 必要 スレッド作成
    GET /messages/threads/:id/messages 必要 メッセージ一覧
    POST /messages/threads/:id/messages 必要 メッセージ送信
    GET /messages/stream?token=<accessToken> 不要 SSE リアルタイム受信

    通知¶

    メソッド パス 認証 概要
    GET /notifications 必要 通知一覧
    POST /notifications/read-all 必要 全件既読
    GET /notifications/stream?token=<accessToken> 不要 SSE リアルタイム受信

    メディアアップロード仕様¶

    POST /posts は multipart/form-data で送信。

    フィールド名 種別 制限
    images ファイル 最大4枚。画像のみ（動画と同時不可）
    video ファイル 1本のみ。画像と同時不可。WebM または MP4
    audio ファイル 1本のみ。WebM/Opus または MP3
    coverImage ファイル 音声投稿のカバー画像（任意）
    body テキスト 投稿本文（最大 1000 文字）
    isNsfw boolean NSFW フラグ
    isAiGenerated boolean AI 生成フラグ

    Magic bytes 検証（MIME 詐称対策）が server-side で行われます。
    動画は video/webm または video/mp4（codecs パラメータなし）で送ること。

    5. リアルタイム通信（SSE）¶

    Plofolia は WebSocket ではなく SSE（Server-Sent Events） を採用しています。

    設計¶

    sequenceDiagram
    participant C as クライアント
    participant API as sol API
    participant DB as PostgreSQL

    C->>API: GET /notifications/stream?token=JWT
    Note right of API: EventSource はカスタムヘッダー非対応<br/>→ JWT をクエリパラメータで渡す
    API->>API: JWT 検証
    API->>API: sseClients.set(userId, response)
    API-->>C: Content-Type: text/event-stream（接続確立）

    loop 25 秒ごと（Nginx タイムアウト対策）
        API-->>C: ": keepalive"
    end

    Note over DB,API: 別ユーザーがいいね
    DB->>API: いいね登録完了
    API->>API: pushNotificationToUser(targetUserId)
    API-->>C: event: notification\ndata: {"type":"like",...}

    Note over C,API: 接続切断
    C--xAPI: 切断
    C->>C: 60 秒後にポーリングへ自動フォールバック
    C->>API: GET /notifications?since=...

    sseClients: Map<userId, Set<Response>> でユーザーごとに複数タブ・複数デバイスの同時接続を管理。

    イベント形式¶

    event: notification
data: {"id":"uuid","type":"like","actorUsername":"...","postId":"...","createdAt":"..."}

event: new_message
data: {"threadId":"uuid","message":{"id":"...","body":"...","senderId":"...",...}}

    6. ユニーク機能の実装概要¶

    タイムコメント¶

    動画・音声の再生位置に紐づいたコメント。

    • timed_comments テーブル: (post_id, user_id, timecode_ms, body)
    • フロントは <video> / <audio> の timeupdate イベントで現在位置を監視
    • timecode_ms ± 3000ms の範囲にあるコメントをオーバーレイ表示
    • VideoPlayer / AudioPlayer は forwardRef + seekTo(ms) ハンドルを公開
    • 全画面時はコメントリストをプレイヤー右横に並べる

    NSFW コンテンツ判定（nsfwjs）¶

    アップロード時に nsfwjs（MobileNetV2 モデル）でブラウザ内推論を実行。

    hentai + porn > 0.6 → 自動フラグ（管理者に通知）
hentai + porn > 0.35 → NSFW フラグを付けるよう提案

    サーバー側では magic bytes によるファイル形式検証のみ行い、 コンテンツ判定はブラウザで完結する設計（サーバー負荷・プライバシーの両立）。

    PWA（Progressive Web App）¶

    • Service Worker: キャッシュファースト（/_next/static/*）+ ネットワークファースト（その他）
    • /api/* はキャッシュ除外（SSE を含むため）
    • Android Chrome でインストールプロンプトが表示される条件をすべて満たす構成

    カーソルページネーション¶

    タイムライン・通知・DM などすべてのリスト API はカーソルベース（?cursor=<id>）。
    ORDER BY created_at DESC, id DESC でソートし、次ページを cursor に渡す。

    8. セキュリティ設計¶

    対策 実装
    パスワード bcrypt（cost factor 10）
    JWT HS256、アクセス 15 分 / リフレッシュ 7 日（httpOnly Cookie）
    リフレッシュローテーション 再使用検出で全セッション即時無効化
    Rate Limiting ログイン 10 req/15min・API 全体 200 req/15min（express-rate-limit）
    アカウントロック ログイン失敗5回で15分ロック
    ファイルアップロード MIME 許可リスト（multer）+ magic bytes 検証（2層）
    メールドメイン検証 登録時に DNS MX レコードを確認
    セキュリティログ security_logs テーブルにログイン・失敗・アップロード拒否を記録
    HTTPS Let's Encrypt（Nginx）
    CSP next.config.mjs で全ページに Content-Security-Policy を付与
    管理画面 HTTP Basic Auth + TOTP MFA + デバイス信頼（30日間スキップ）
    セキュリティヘッダー HSTS・X-Frame-Options・X-Content-Type-Options 等を Nginx で付与

    9. フィードバック・バグ報告¶

    アプリ内から報告（推奨）¶

    • サイドバーの「バグを報告」ボタン → カテゴリ・説明・スクリーンショット（最大3枚）
    • エラー境界の画面（白くなったとき）から直接送信

    報告内容に含めると助かるもの¶

    • 発生した操作手順
    • ブラウザ・OS の情報
    • エラーメッセージ（コンソールに表示されていれば）
    • 期待していた動作 vs 実際の動作

    技術的なフィードバック（設計・API の提案など）は /contact のお問い合わせフォームへ。
