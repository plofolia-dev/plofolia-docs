    2026-04-30（セッション6）¶

    MFA デバイス信頼機能（AWS方式・30日間スキップ）実装¶

    Aurora MFA の「毎回コードを求められる」問題を解消するため、AWS方式のデバイス信頼を実装。

    仕組み¶

    1. MFA 認証完了後、バックエンドが 64-char hex のランダムトークンを生成し SHA-256 ハッシュを DB に保存（30日有効）
    2. raw トークンをクライアントに返却 → Aurora が aurora-device-token + aurora-device-token-expiry として localStorage に保存
    3. 次回ログイン時、localStorage からトークンを読み取りパスワードと一緒に送信
    4. バックエンドで DB のハッシュと照合 → 一致すれば MFA をスキップしてアクセストークンを直接発行

    実装内容¶

    • DB: admin_trusted_devices テーブル追加（migration 0038_admin_trusted_devices.sql・EC2 DB 適用済み）
    • カラム: id, user_id, token_hash, expires_at, created_at
    • インデックス: user_id + token_hash 両方
    • sol admin.auth.ts: 全面書き直し
    • adminLogin(identifier, password, deviceToken?): 有効な deviceToken が DB にあれば MFA をスキップ
    • verifyMfaLogin(): TOTP 検証後に admin_trusted_devices へ INSERT し、raw トークンをレスポンスに含める
    • disableMfa(): MFA 無効化時に DELETE FROM admin_trusted_devices WHERE user_id = ... で全デバイストークンを削除
    • 期限切れトークンは adminLogin 時に非同期でクリーンアップ（fire-and-forget）
    • sol admin-app.ts: loginSchema に deviceToken: z.string().optional() を追加し adminLogin に渡す
    • aurora api/auth/route.ts: 第1ステップで deviceToken を admin API に転送、第2ステップで deviceToken をレスポンスに含めて返却
    • aurora login/page.tsx:
    • useEffect でマウント時に localStorage のトークンを読み取り（期限チェック付き）
    • ステップ1送信時に deviceToken を body に含める
    • MFA 認証成功後に data.deviceToken を localStorage に30日有効期限で保存

    デプロイ¶

    • cf3e701 でコミット・push
    • EC2: npm run build → pm2 restart plofolia-sol plofolia-nova → migration 0038 適用 → aurora npm run build + pm2 restart plofolia-aurora

    #3 ファイルアップロード magic bytes 検証強化¶

    • 既存の画像のみ検証を動画・音声まで拡張（sol/src/lib/sanitize.ts）
    • 追加タイプ:
    • video/webm / audio/webm: 1A 45 DF A3（EBML ヘッダー）
    • video/mp4 / video/quicktime: offset4 66 74 79 70（ftyp ボックス）
    • audio/mpeg（MP3）: FF FB / FF F3 / FF F2 / ID3 の4パターン
    • audio/wav: RIFF + offset8 WAVE（AND 条件）
