    Plofolia — プロジェクト概要¶

    関係者への紹介・説明用サマリ。詳細な技術仕様は各 docs/ ファイルを参照。

    命名由来¶

    Plofolia = Plot + Portfolia

    Plot（プロット）¶

    創作の途中段階、ラフ、WIP（Work In Progress）を投稿する SNS 的な側面を表します。 クリエイターが日々の制作過程をタイムラインに気軽に投稿し、フォロワーと交流する場です。

    Portfolio（ポートフォリオ）¶

    完成作品を整理して公開するポートフォリオとしての側面を表します。 シリーズ機能やプロフィールページを使って、代表作を閲覧者に見せるショーケースとして機能します。

    コンセプト¶

    クリエイターにとって SNS（Plot）とポートフォリオサイト（Portfolio）を一か所で管理できる プラットフォームを目指しています。 日々の WIP を気軽に投稿しながら、同時に完成作品をポートフォリオとして整理・公開できます。

    閲覧者にとっても、好みのクリエイターの制作過程を楽しみながら代表作にアクセスできる、 クリエイターとファンの距離が近い場所を提供します。

    プロダクトコンセプト¶

    Plofolia（プロフォリア） は、クリエイター向けの SNS ＋ ポートフォリオプラットフォームです。 NSFW コンテンツおよび AI 生成コンテンツを含む、あらゆる表現スタイルのクリエイターが 作品を発表・整理・収益化できる専用の場所を提供します。

    既存の SNS は汎用向けに設計されており、クリエイターの活動にとって最適ではない制約や 不透明なアルゴリズムを抱えています。Plofolia はクリエイターを中心に据えた設計で、 投稿の可視性・作品の整理・ファンとの交流を、クリエイターにとって自然な形で実現します。

    AI 時代のミッション¶

    AI が生成コンテンツの制作コストを急速に下げる時代において、Plofolia は二つの軸でクリエイターを支えます。

    人間クリエイターへ: WIP 投稿という制作過程の記録が「人間が作った」ことの証跡となり、希少価値を可視化します。AI 学習拒否宣言・将来的なクリエイター集団としての権利交渉など、AI 時代を生き抜くための仕組みを継続的に整備します。

    AI クリエイターへ: AI 生成作品を他の投稿と同等に扱い、アルゴリズムによる差別はしません。タイムラインの AI フィルターで AI 作品を好むファンへ直接届く仕組みを提供し、プロンプト試行・スタイル探求をWIP として堂々と公開できる場所です。AI を使った創作もれっきとした表現活動として尊重します。

    差別化ポイント（強み）¶

    1. 透明なアルゴリズム — 投稿は意図せず非表示になりません。クリエイター分析ダッシュボードで投稿のリーチ状況を自分で確認できます
    2. タイムコメント機能 — 動画・音声投稿に時刻付きコメントを付けられる（YouTube ライブ風オーバーレイ）。他のプラットフォームにない、動画・音声作品へのエンゲージメント体験
    3. 表現に対するオープンな姿勢 — NSFW コンテンツや AI 生成作品を制限・差別しない方針。AI 学習拒否宣言機能も提供し、クリエイターの権利を尊重
    4. 制作スタイル別タイムライン — 画像 / 映像 / 音声 の3ドメインタブに分離し、さらに人間 / AI フィルターで絞り込める。AI クリエイターは AI ファンへ直接届き、人間クリエイターは人間作品を求めるファンへ届く
    5. SNS ＋ ポートフォリオの一体設計 — WIP 投稿を完成後に「作品として整理」で紐づけるだけで、制作過程のツリーが付いた作品ポートフォリオが完成。シリーズ機能と合わせて制作活動の全体をひとつの場所で公開できる
    6. 人間クリエイターの権利保護 — WIP 投稿の積み重ねが制作過程の証跡となり、「人間が作った」という事実を可視化します。AI 学習拒否宣言・将来的なクリエイター集団としての権利交渉など、AI 時代を生き抜くための仕組みを継続的に整備します
    7. AI クリエイターへの公平な舞台 — プロンプト試行・スタイル探求をWIP として公開できる場所。AI 生成作品はアルゴリズムで差別されず、AI ファンフィルターで的確なオーディエンスへ届きます

    ターゲット¶

    一次ターゲット（投稿者）¶

    人間クリエイター:

    • 投稿が埋もれず確実に届く場所を求めているイラストレーター・漫画家・映像作家・音声クリエイター
    • 成人向け作品を適切な環境で発表したい漫画家・コスプレイヤー・写真家
    • WIP から完成作品まで制作活動をひとつのプロフィールで整理・公開したいクリエイター
    • AI の普及によって自分の作品・仕事の価値が脅かされていると感じており、制作過程を記録して「自分が作った」ことを示したいクリエイター
    • 動画・音声作品のファンとタイムコメントでリアルタイムに交流したいクリエイター

    AI クリエイター:

    • AI 生成コンテンツを制限なく発表できる場所を求めているクリエイター（イラスト・映像・音声など、あらゆるジャンルの AI 活用クリエイター）
    • プロンプトエンジニアリング・スタイル探求・編集工程を透明に公開し、AI 創作の価値を伝えたいクリエイター
    • 他のプラットフォームで差別・垢BAN のリスクを感じており、安心できる居場所を求めているクリエイター
    • AI コンテンツを好むファンに直接届く専用フィルターを活用したいクリエイター

    二次ターゲット（閲覧者）¶

    • 好みのジャンル・国籍・スタイルでフィルタリングしてクリエイターを探したいユーザー
    • タイムコメントで推しの動画・音声作品に参加したいファン

    主要機能¶

    カテゴリ 機能
    タイムライン 画像 / 映像 / 音声 の3タブ分離 + 人間/AI フィルター、地域フィルタリング
    投稿 画像（最大 4 枚）・動画・音声（カバー画像付き）対応
    ポートフォリオ作品 WIP 投稿を完成作品に紐づけて制作過程をツリー表示。「作品として整理」で PostCard メニューから作成
    クリエイター分析 投稿リーチ・フォロワー推移ダッシュボード
    タイムコメント 動画・音声に時刻付きコメント（YouTube ライブ風オーバーレイ）
    シリーズ投稿 複数作品をシリーズにまとめてプロフィールに表示
    多言語対応 日本語・英語・中国語・韓国語・スペイン語（5 言語）
    AI 学習拒否 設定画面からワンクリックで AI 学習拒否宣言・プロフィールにバッジ表示
    DM リアルタイムチャット（SSE）
    通知 リアルタイム通知（SSE）
    ミュート/ブロック ユーザー単位で非表示・ブロック、タイムラインから自動除外
    管理システム ユーザー BAN・投稿削除・監査ログ・バグ報告管理（Aurora 管理画面）
    招待コード制 クローズドベータ期間中は招待コード制で登録を制限（設定で切替可）

    技術構成（概要）¶

    層 技術
    フロントエンド Next.js 14 App Router + TypeScript + Tailwind CSS
    バックエンド Node.js + Express + TypeScript + Drizzle ORM
    データベース PostgreSQL 16
    インフラ AWS EC2（t3.micro / ap-northeast-1）+ Nginx + PM2
    認証 JWT（アクセス 15 分 + リフレッシュ 7 日）+ Google/X OAuth
    ストレージ AWS S3（ap-northeast-1、plofolia-media バケット）
    CI/CD GitHub Actions + Slack 通知（#dev-alerts）
    コスト 現在 $20〜30/月。詳細は docs/ops/COST.md 参照

    ビジネスモデル¶

    現在（パイロット期間中）¶

    支援リンクは非表示にしている（NEXT_PUBLIC_SUPPORT_ENABLED=false）。

    非表示にした経緯:

    • Ko-fi（国際向け寄付）・Ofuse（日本向け寄付）・Fantia を検討
    • いずれも Stripe 経由の本人確認が必要
    • Stripe Japan 審査には 特定商取引法に基づく表記ページ がサイト上に必要（Issue #69）
    • Fantia は NSFW コンテンツに馴染みが強く、ブランドイメージに合わない
    • パイロット期間はユーザー体験の検証に集中し、決済対応は後回しにする方針に決定

    復元手順（Issue #70）:

    1. 特定商取引法ページを terra に追加する（Issue #69 完了後）
    2. Stripe Japan 審査を通過する
    3. EC2 terra/.env.local の NEXT_PUBLIC_SUPPORT_ENABLED=false を削除する
    4. npm run build → pm2 restart plofolia-terra --update-env

    関係者を増やす際のアクセス権限方針¶

    原則: 業務に必要な最小権限のみ付与。

    関係者 付与するアクセス 付与しないもの
    モデレーター Aurora モデレータアカウント（投稿削除・報告対応・問い合わせ返信） BAN/ユーザー削除・招待コード・監査ログ・GitHub・AWS
    副業エンジニア GitHub（develop ブランチへの PR）・ローカル環境 本番 AWS 認証情報・sol/.env の機密情報
    正社員エンジニア GitHub（全ブランチ）・AWS（必要なサービスのみ） ルートアカウント・支払い情報
    法務アドバイザー ドキュメント（docs/policy/）のみ コード・DB・ユーザーデータ

    モデレータアカウントの発行: Aurora /accounts（admin のみ）からユーザー名・メールアドレス・パスワードを入力して作成。
