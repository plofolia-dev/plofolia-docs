    コスト管理¶

    Plofolia の AWS・外部サービス費用の把握・予測・最適化まとめ。

    現在の月額コスト（2026年4月時点）¶

    サービス プラン 月額（概算） 備考
    AWS EC2 t3.micro (ap-northeast-1) ~$10 オンデマンド。Reserved 予約で約40%削減可
    AWS EBS gp3 20GB（EC2のルートボリューム） ~$1.60 -
    AWS S3 ストレージ〜1GB ~$0.03 現在はローカルストレージ。S3未使用
    AWS SES 〜1,000通/月 ~$0.10 送信1,000通まで無料枠あり
    ドメイン plofolia.com（Namecheap等） ~$1（年間$12） 更新費用
    Let's Encrypt SSL証明書 $0 無料。90日ごとに自動更新
    GitHub Free プラン $0 プライベートリポジトリ1つ
    Cloudflare Tunnel 無料プラン $0 ローカル開発用（本番不使用）
    Ko-Fi 無料プラン（寄付5%手数料） $0（手数料のみ） 手数料は受け取り額から差し引き
    合計 ~$13/月 ユーザー0人でも発生する固定費

    フェーズ別コスト予測¶

    Phase 1: 現在〜小規模（〜1,000人）¶

    EC2 t3.micro:              $10/月
EBS 20GB gp3:               $2/月
S3（〜10GB）:               $0.25/月
SES（〜5,000通/月）:        $0.50/月
ドメイン:                   $1/月
────────────────────────────────
合計:                      ~$14/月

    ユーザーが増えても EC2とS3 しか増えない。1,000人規模ではt3.microで十分。

    Phase 2: 中規模（1,000〜10,000人）¶

    EC2 t3.small（スケールアップ）: $20/月
EBS 50GB:                         $4/月
S3（〜100GB）:                    $2.50/月
SES（〜20,000通/月）:             $2/月
CloudFront（CDN、動画配信）:      $5〜$20/月
ドメイン:                         $1/月
────────────────────────────────────────
合計:                            ~$35〜$50/月

    動画・音声が多いと S3とCloudFront の転送コストが支配的になる。

    Phase 3: 大規模（10万人〜）¶

    EC2 t3.medium × 2（ALB配下）:    $80/月
RDS db.t3.small Multi-AZ:        $60/月
ElastiCache（Redis）t3.micro:    $25/月
S3（〜1TB）:                     $24/月
CloudFront（〜10TB転送）:        $90/月
ALB:                              $20/月
SES（〜100,000通/月）:           $10/月
────────────────────────────────────────
合計:                            ~$310/月

    この規模になるとチップ手数料10%の収益（月数十万円）でコストをカバーできる見込み。

    S3 メディアコスト詳細¶

    現在はローカルストレージのため S3 費用はほぼゼロ。STORAGE_PROVIDER=s3 に変更した時点で発生。

    料金体系（ap-northeast-1 / 東京リージョン）¶

    項目 料金 備考
    ストレージ $0.025/GB/月 標準クラス
    PUT/COPY リクエスト $0.005/1,000リクエスト 投稿アップロード時
    GET リクエスト $0.0004/1,000リクエスト 画像表示時
    EC2→S3 転送 無料 同リージョン内
    S3→インターネット $0.114/GB（最初の10TB） CloudFront経由なら安くなる
    STANDARD-IA $0.0138/GB/月 メディアバックアップに最適（54%削減）

    実際の見積もり例¶

    ユーザー1,000人 × 平均10投稿 × 画像2MB = 20GB
  ストレージ: 20GB × $0.025 = $0.50/月
  転送（表示）: 月100GB想定 × $0.114 = $11.40/月
  → CloudFront導入で転送コストを60〜80%削減可能

    結論: ユーザーが増えると S3 転送費用が支配的になる。1,000人規模で CloudFront 導入を検討。

    SES メール費用¶

    利用量 料金 備考
    月1,000通以下 無料 EC2から送信の場合
    1,001通〜 $0.10/1,000通 非常に安い
    添付ファイル $0.12/GB テキストメールのみなら無関係

    メール用途（認証・通知）では月数万円かかることはまずない。

    コスト監視・アラート設定¶

    AWS Budgets でアラートを設定する（必須）¶

    AWSコンソール → Billing → Budgets → Create budget
  タイプ: Cost budget
  金額: $50/月（現状の4倍で設定）
  通知: 80%到達時と100%到達時にメール送信

    Cost Explorer で確認する¶

    AWSコンソール → Cost Management → Cost Explorer
  日次グラフ → サービス別コストを確認
  Filter: サービスごとに分解（EC2/S3/SES）

    EC2 毎月の確認コマンド¶

    # EC2インスタンスの現在の料金タイプを確認
aws ec2 describe-instances --instance-ids i-04f92f1a7aad49ef2 \
  --query 'Reservations[0].Instances[0].InstanceLifecycle'

    コスト最適化 Tips¶

    今すぐできること¶

    最適化 効果 実施方法
    EC2 Reserved Instance 購入 約40%削減 AWSコンソール → EC2 → Reserved Instances → 1年契約
    EBS スナップショット管理 $0.05/GB/月削減 古いスナップショットを削除（月次で確認）
    S3 Intelligent-Tiering 設定 〜50%削減 アクセスパターン不明なファイルに自動適用
    CloudWatch Logs 保持期間 微小 デフォルト「無期限」→ 30日に変更

    将来の最適化¶

    最適化 効果 タイミング
    CloudFront 導入 転送コスト60〜80%削減 S3移行後・ユーザー1,000人超
    EC2 Savings Plans 〜20%削減 利用量が安定してから
    S3 STANDARD-IA 移行 ストレージ54%削減 30日間アクセスのないファイル
    ALB + Auto Scaling 導入 無駄な常時費用削減 10,000人超
