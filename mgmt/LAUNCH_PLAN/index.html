    Plofolia ローンチ計画¶

    リリース前の準備チェックリスト・ユーザー獲得戦略・マイルストーン。

    ローンチ準備チェックリスト¶

    [Critical] ローンチ前に必須¶

    カテゴリ 項目 状況 備考
    メール AWS SES Sandbox解除 未対応 Sandbox のままだと認証メールが検証済みアドレスにしか届かない。AWSコンソール → SES → Request production access
    メール 送受信メールアドレス整備 未対応 dmca@plofolia.com / privacy@plofolia.com を実際に受信できる状態にする（Google Workspace $6/月 or Zoho Mail 無料）
    監視 Uptime 監視 未対応 UptimeRobot（無料）で https://plofolia.com/health を5分ごとに死活監視。ダウン時にメール通知
    AWS Budgets アラート 未対応 $50/月・80%/100% でメール通知。ユーザータスク（AWSコンソールで3分）
    法務 DMCA 指定代理人登録 未対応 米国著作権局に登録（$6/年）。NSFW サービスには特に重要

    [Important] ローンチ後早期に対応¶

    カテゴリ 項目 状況 備考
    エラー監視 Sentry 導入 未対応 本番エラーをリアルタイム把握。無料プランで十分（5000 errors/月）。sol/terra 両方に設定
    パフォーマンス OGP 画像 CDN 未対応 S3 + CloudFront で画像配信（現在 EC2 ローカル保存）。ユーザーが増えてから

    [Done] 実装済み¶

    • HTTPS / SSL（Let's Encrypt）
    • Rate Limiting（ログイン 10回/15分・API 200回/15分）
    • セキュリティヘッダー（HSTS・X-Frame-Options 等）
    • メール認証フロー（未認証はログイン不可）
    • バグ報告システム（ユーザー送信 + Aurora 管理）
    • DB 自動バックアップ（毎日 3時 → S3）
    • CI/CD（GitHub Actions + PM2）
    • OGP metadata（投稿詳細ページ）
    • robots.txt（NSFW 投稿・認証系ページのクロール制御）
    • sitemap.xml（静的5言語ページ + トレンドタグの動的生成）
    • プライバシーポリシー・利用規約・DMCA ページ
    • DMCA 申告フォーム（DB 保存）

    ユーザー獲得戦略¶

    ターゲットユーザー¶

    一次ターゲット（最初の 500 人）:

    • 投稿が確実に届く場所を求めているイラストレーター・漫画家・映像作家
    • AI 生成コンテンツを制限なく発表したいクリエイター（AI 絵師・AI 映像作家）
    • WIP から完成作品まで制作活動を一か所でまとめて公開したいクリエイター

    二次ターゲット（500 ～ 5000 人）:

    • タイムコメントで動画・音声作品のファンとリアルタイムに交流したいクリエイター
    • AI 学習拒否を明示したいクリエイター
    • 閲覧専用ユーザー（作品ファン・ポートフォリオを探している人）

    チャネル別戦略¶

    1. SNS（X / Bluesky / Mastodon 等）— 最優先

    メッセージ: 「クリエイターの制作活動をまるごと支える SNS を作った」

    • 創設者アカウント（個人または @plofolia）でローンチ告知
    • タイムコメント・ポートフォリオ整理などの独自機能を実演・紹介
    • イラストレーター・映像作家・音声クリエイターのインフルエンサーへのアプローチ
    • 「招待コード配布投稿」でバイラル拡散（招待制の仕組みを逆手に取る）

    2. Pixiv / Skeb — 二次流入

    • Pixiv プロフィールに Plofolia URL を記載してもらうよう早期ユーザーに依頼
    • Skeb クリエイターへのアプローチ（ポートフォリオ用途として）

    3. Discord — コミュニティ構築

    • NSFWクリエイターコミュニティサーバーへの告知
    • r/StableDiffusion, r/AIArt（Reddit）への投稿（ルール確認必須）
    • Plofolia 公式 Discord サーバー開設（フィードバック窓口）

    4. Mastodon / Misskey — 移住勢への訴求

    • X に不満を持つクリエイターが流入しているコミュニティ
    • mstdn.jp, misskey.io 等のタグ活用

    ローンチフェーズ別施策¶

    Phase 1（招待制ベータ: ～100 人）

    • 招待コードは限定配布（信頼できるクリエイター・知人）
    • 目的: バグ・UX 問題を潰す。コアフィードバックを得る
    • KPI: DAU/MAU 比 30% 以上、バグ報告数 < 5件/日

    Phase 2（限定公開: 100 ～ 1000 人）

    • 招待制のまま X でコード配布ツイート
    • OGP 画像でシェアが映えることを活用（#Plofolia ハッシュタグ）
    • KPI: 投稿率 40% 以上（登録ユーザーの 4 割が 1 投稿以上）

    Phase 3（フルオープン: 1000 人～）

    • 招待制解除
    • インフルエンサーへの正式アプローチ
    • プレスリリース（ITmedia・AUTOMATON 等のゲーム/クリエイター系メディア）

    マイルストーン¶

    Phase 0 — ローンチ準備（2週間以内）¶

    目標: 本番環境を「招待制ベータ」として公開できる状態にする

    期限 担当 タスク 状況
    今週中 ユーザー AWS SES Sandbox 解除申請（審査に数日かかることがある） 未
    今週中 ユーザー メール受信環境整備（Google Workspace or Zoho Mail） 未
    今週中 ユーザー AWS Budgets アラート設定 未
    今週中 ユーザー UptimeRobot 設定（plofolia.com/health） 未
    完了済み Claude robots.txt 作成・デプロイ（2026-04-29） 完了
    完了済み Claude sitemap.xml 作成（静的5言語 + トレンドタグ動的生成） 完了
    来週中 ユーザー DMCA 指定代理人登録（copyright.gov） 未
    来週中 Claude Sentry 導入（sol + terra） 未

    Phase 1 — 招待制ベータ（1ヶ月）¶

    目標: 招待制で 50～100 人に利用してもらい、コアループを検証する

    • 招待コードを信頼できるクリエイター 10～20 人に配布
    • バグ報告・フィードバックを週次でトリアージ
    • 投稿・いいね・フォローのコアループ動作確認
    • 主要バグが出尽くしたら Phase 2 へ移行

    成功指標:

    • 登録者 50 人以上
    • 週次 DAU 30% 以上
    • Critical バグ 0

    Phase 2 — 限定公開（2～3ヶ月）¶

    目標: 招待コード配布でバイラル拡散、1000 ユーザー獲得

    • 招待コード配布投稿を週 1 回程度
    • OGP / シェア体験の最終調整
    • タイムコメント・シリーズ機能の認知拡大
    • コンテンツモデレーション対応準備

    成功指標:

    • 登録者 1000 人
    • 月間 100 投稿以上
    • SNS 言及数週次増加

    Phase 3 — フルオープン（3～6ヶ月後）¶

    目標: 招待制解除・収益化検討開始

    • 招待コード制を解除してオープン登録
    • Ko-Fi 以外の収益モデル検討（クリエイター支援・サブスク）
    • 必要に応じて Stripe / CCBill 実装
    • CloudFront 導入（帯域増加対応）
    • チームメンバー（副業エンジニア）検討

    成功指標:

    • 登録者 5000 人以上
    • Ko-Fi 収益 > AWS コスト（黒字化）

    最優先アクション（今すぐ）¶

    1. AWS SES Sandbox 解除 — メール認証フローが本番で使えない。申請から審査まで数日かかるため先に動く
    2. robots.txt — NSFW コンテンツのクロール制御。5分で実装できる
    3. 受信メールアドレス整備 — dmca@plofolia.com への通知が届かない状態は NG
    4. UptimeRobot — 無料・5分設定。ダウン検知がなければ本番運用リスクあり
