    Plofolia メール設定ガイド¶

    support@plofolia.com の送受信環境構築手順。

    構成概要¶

    役割 サービス 備考
    受信（support@ 宛を受け取る） Google Workspace MX レコードで転送
    手動返信（サポート対応） Gmail（Workspace） support@plofolia.com として送受信
    システム送信（認証メール等） AWS SES MAIL_FROM=support@plofolia.com

    設定済み（2026-04-30 完了）:

    • Google Workspace Business Starter 契約済み（$6/月）
    • support@plofolia.com ユーザー作成済み
    • お名前.com DNS 全レコード設定済み・反映確認済み
    • EC2 sol/.env の MAIL_FROM=support@plofolia.com 変更済み

    DNS レコード（お名前.com 設定済み）¶

    タイプ ホスト名 値 用途
    MX @ aspmx.l.google.com（優先度1） Gmail 受信
    MX @ alt1-4.aspmx.l.google.com（優先度5/10） Gmail 受信（冗長）
    TXT @ v=spf1 include:_spf.google.com include:amazonses.com ~all SPF
    TXT google._domainkey v=DKIM1; k=rsa; p=MIIB... DKIM（Gmail）
    CNAME 5z7pwem4...._domainkey *.dkim.amazonses.com DKIM（SES × 3）
    TXT _dmarc v=DMARC1; p=none; DMARC
    TXT @ google-site-verification=oE5o0u... ドメイン所有確認
    MX mail feedback-smtp.ap-northeast-1.amazonses.com（優先度10） SES バウンス処理
    TXT mail v=spf1 include:amazonses.com ~all SES 送信元認証

    Google Workspace セットアップ手順（参考・完了済み）¶

    1. workspace.google.com で Business Starter に申し込み
    2. ドメイン所有確認: お名前.com に google-site-verification=... TXT レコードを追加
    3. support@plofolia.com ユーザーを作成
    4. Gmail 有効化: お名前.com に MX レコード 5件を追加
    5. DKIM 設定: Admin Console → Apps → Gmail → メール認証 → DKIM レコードを生成 → お名前.com に追加 → 「認証を開始」
    6. SPF 設定: お名前.com に TXT レコード追加（Google + SES 両方含める）

    注意点:

    • ドメインホストはお名前.com（*.dnsv.jp）。Route 53 ではない
    • Google Workspace セットアップ時に「ドメインはすでに使用されています」と表示されたが、管理者パスワード再設定フローで解決
    • DKIM ホスト名は google._domainkey（ルートドメイン @ に設定すると反映されない）

    sol の MAIL_FROM（設定済み）¶

    EC2 の sol/.env:

    MAIL_FROM=support@plofolia.com

    変更後に pm2 restart plofolia-sol plofolia-nova 実施済み。

    今後の課題¶

    SES Sandbox 解除¶

    本番環境で任意アドレスへのシステムメール送信には SES の本番アクセスリクエストが必要。

    手順: AWS コンソール → Amazon SES → Account dashboard → Request production access

    将来実装: メール → Aurora チケット自動連携¶

    現状は Gmail（support@plofolia.com）と Aurora（contact_inquiries テーブル）を別々に管理している。未ログインユーザーが返信メールを送ってきた場合、Aurora のスレッドには反映されない。

    設計方針¶

    採用アプローチ: Google Apps Script（無料・低コスト）

    Google Workspace の Gmail トリガーを使い、受信メールを自動的に Aurora の contact_inquiries スレッドに取り込む。

    未ログインユーザー
  → support@plofolia.com へ返信メール送信
  → Gmail が受信
  → Google Apps Script が onMessage トリガーで検知
  → Sol の管理 API（POST /admin/contact-inquiries/:id/mail-reply）を呼び出す
  → contact_messages テーブルにメッセージを挿入（sender_type: 'user'）
  → Aurora のスレッド画面に反映

    実装ステップ¶

    1. Sol 側: メール受信用 Webhook エンドポイントを追加¶

    POST /admin/contact-mail-reply（管理 API）:

    // リクエスト body
{
  inquiryId: string,   // メールの件名や Reference ヘッダから抽出
  from: string,        // 送信者メールアドレス（照合用）
  content: string,     // メール本文（テキスト部分）
  secret: string       // Google Apps Script 共有シークレット
}
    • secret で Google Apps Script からのリクエストのみ受け付け（環境変数 MAIL_WEBHOOK_SECRET）
    • inquiryId の照合: 件名に [#ID] 形式でチケット番号を埋め込む（返信メールの件名に残る）
    • メールアドレスが inquiry.email と一致するか確認
    • contact_messages に sender_type: 'user' で挿入

    2. 返信メールの件名にチケット番号を埋め込む¶

    buildContactReplyEmail の subject を変更:

    【Plofolia】お問い合わせへのご返信: {subject} [#{inquiryId の先頭8文字}]

    ユーザーが返信すると Re: 【Plofolia】お問い合わせへのご返信: ... [#abc12345] という件名になり、Apps Script がここから ID を抽出できる。

    3. Google Apps Script を設定¶

    function onNewEmail(e) {
  const thread = e.thread
  const message = thread.getMessages().slice(-1)[0]
  const subject = message.getSubject()
  const body = message.getPlainBody()
  const from = message.getFrom()

  // 件名から inquiryId を抽出
  const match = subject.match(/\[#([a-f0-9]+)\]/)
  if (!match) return // Plofolia 以外のメールは無視

  const inquiryId = match[1]

  UrlFetchApp.fetch('https://plofolia.com/api/v1/admin/contact-mail-reply', {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify({
      inquiryId,
      from,
      content: body,
      secret: PropertiesService.getScriptProperties().getProperty('WEBHOOK_SECRET'),
    }),
  })
}
    • Gmail のフィルタ: support@plofolia.com 宛のみ対象
    • トリガー: Apps Script の「編集」→「現在のプロジェクトのトリガー」→ Gmail トリガーを追加

    実装優先度¶

    将来（ユーザーが増えてサポート量が増えてから）

    現状はユーザー数が少なく、Gmail と Aurora を個別に見る運用で十分。未ログインユーザーからのメール返信は Gmail で直接確認・返信する。実装コストより他の優先事項を先に進める。
