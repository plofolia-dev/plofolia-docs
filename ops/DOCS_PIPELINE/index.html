    ドキュメント自動公開パイプライン¶

    docs/ フォルダに Markdown を書くだけで、GitHub Pages に自動デプロイされる仕組みの解説。

    全体像¶

    graph TD
    A[ローカル: docs/*.md を編集] --> B[git push origin develop]
    B --> C{GitHub Actions\nDeploy docs to plofolia-docs}
    C --> D[MkDocs Material\nでビルド]
    D --> E[site/ に静的 HTML 生成]
    E --> F[plofolia-docs リポジトリ\nmain ブランチにプッシュ]
    F --> G[GitHub Pages\n自動公開]
    G --> H[https://plofolia-dev.github.io/plofolia-docs/]

    リポジトリ構成¶

    plofolia/          ← プライベートリポジトリ（コード + docs/）
├── mkdocs.yml     ← MkDocs の設定ファイル
├── docs/
│   ├── index.md           ← トップページ
│   ├── user/
│   │   ├── USER_GUIDE.md       ← 一般ユーザー向け
│   │   └── ENGINEER_GUIDE.md   ← エンジニア向け
│   └── ops/
│       └── ...（内部運用ドキュメント、公開しない）
└── .github/workflows/
    └── docs.yml   ← デプロイ自動化ワークフロー

plofolia-docs/     ← パブリックリポジトリ（ビルド済み HTML のみ）
└── main ブランチ  ← GitHub Actions が自動生成・更新する（直接編集しない）

    デプロイフロー（詳細）¶

    sequenceDiagram
    participant Dev as 開発者
    participant Priv as plofolia（private）
    participant GA as GitHub Actions
    participant Pub as plofolia-docs（public）
    participant GH as GitHub Pages

    Dev->>Priv: git push origin develop<br/>（docs/ または mkdocs.yml を変更）
    Priv->>GA: workflow トリガー
    GA->>GA: actions/checkout@v4<br/>リポジトリをチェックアウト
    GA->>GA: pip install mkdocs-material
    GA->>GA: mkdocs build<br/>→ site/ に HTML 生成
    GA->>Pub: peaceiris/actions-gh-pages<br/>site/ を main ブランチにプッシュ
    Pub->>GH: main ブランチ更新を検知
    GH->>GH: 静的ファイルを配信開始
    Note over GH: https://plofolia-dev.github.io/plofolia-docs/

    トリガー条件¶

    .github/workflows/docs.yml の on: に定義されている。

    トリガー 条件
    自動（push） develop または main ブランチへのプッシュで、かつ docs/** / mkdocs.yml / docs.yml のいずれかが変更されたとき
    手動 GitHub → Actions → Deploy docs to plofolia-docs → Run workflow

    ポイント: sol/ や terra/ だけの変更ではドキュメントビルドは走らない（パスフィルタで除外）。

    使用技術¶

    技術 役割
    MkDocs Material Markdown → 静的 HTML サイトに変換。Mermaid 図・検索・ダークテーマ対応
    GitHub Actions push をフックにビルド→デプロイを自動実行
    peaceiris/actions-gh-pages ビルド成果物を別リポジトリの指定ブランチにプッシュするアクション
    DOCS_DEPLOY_PAT plofolia（private）から plofolia-docs（public）への書き込みを許可する Personal Access Token
    GitHub Pages plofolia-docs の main ブランチを HTTPS で公開

    ページを追加する手順¶

    1. Markdown ファイルを作成¶

    # 例: よくある質問ページを追加
touch docs/user/FAQ.md

    2. mkdocs.yml の nav に追加¶

    nav:
  - ホーム: index.md
  - ユーザーガイド: user/USER_GUIDE.md
  - エンジニア向け技術解説: user/ENGINEER_GUIDE.md
  - よくある質問: user/FAQ.md # ← 追加

    3. push するだけで自動公開¶

    git add docs/user/FAQ.md mkdocs.yml
git commit -m "docs: FAQ ページ追加"
git push origin develop

    数分後に https://plofolia-dev.github.io/plofolia-docs/ に反映される。

    トラブルシューティング¶

    Actions が動かない¶

    • docs/・mkdocs.yml・docs.yml 以外のファイルしか変更していない場合はスキップされる
    • GitHub → Actions → Deploy docs to plofolia-docs → Run workflow で手動実行できる

    DOCS_DEPLOY_PAT エラー¶

    • PAT の有効期限切れ・スコープ不足の可能性
    • GitHub → Settings → Developer settings → Personal access tokens で PAT を再生成し、plofolia リポジトリの Secrets に DOCS_DEPLOY_PAT を上書き登録する

    Mermaid が表示されない¶

    • mkdocs.yml の pymdownx.superfences の設定が消えていないか確認する
    • ローカルでプレビューする場合: pip install mkdocs-material && mkdocs serve（http://localhost:8000 で確認）
