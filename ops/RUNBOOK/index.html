    1. サイトに繋がらない（全面ダウン）¶

    https://plofolia.com → 到達不能

    確認手順¶

    Step 1: EC2インスタンスが起動しているか

    AWSコンソール → EC2 → インスタンス → i-04f92f1a7aad49ef2 のステータスを確認。
    停止していれば「インスタンスを起動」を押す。

    Step 2: EC2にSSH接続できるか

    # ローカルから（PowerShell）
$aws = "C:\Program Files\Amazon\AWSCLIV2\aws.exe"
$pubKey = Get-Content "E:\DEV\plofolia\ec2_key.pub" -Raw
& $aws ec2-instance-connect send-ssh-public-key `
  --instance-id i-04f92f1a7aad49ef2 `
  --availability-zone ap-northeast-1a `
  --instance-os-user ubuntu `
  --ssh-public-key $pubKey

    ssh -i /e/DEV/plofolia/ec2_key -o StrictHostKeyChecking=no ubuntu@13.115.254.4

    接続できなければ → セキュリティグループでポート22が開いているか確認。

    Step 3: PM2サービスの状態確認

    pm2 status

    errored または stopped のサービスがあれば:

    pm2 logs <service-name> --lines 50 --nostream
pm2 restart <service-name>

    Step 4: Nginxの状態確認

    sudo systemctl status nginx
sudo nginx -t
sudo systemctl reload nginx

    Step 5: ディスク容量確認

    df -h
# /dev/root が 95%以上 → ログやキャッシュを削除
pm2 flush              # PM2ログをクリア
npm cache clean --force  # npmキャッシュ削除

    2. 502 Bad Gateway¶

    Nginxは動いているが、バックエンドに接続できない状態。

    確認手順¶

    Step 1: ポートがlistenしているか確認

    # sol（メインAPI）
curl -s http://127.0.0.1:4000/health   # → {"status":"ok"} でなければ異常

# terra（フロントエンド）
curl -s http://127.0.0.1:3000          # → HTMLが返れば正常

# nova（管理API）
curl -s http://127.0.0.1:4001/health

    Step 2: PM2ログで原因特定

    pm2 logs plofolia-sol --lines 100 --nostream
pm2 logs plofolia-terra --lines 100 --nostream

    Step 3: よくある原因と対処

    症状 原因 対処
    sol が起動失敗 npm run build されていない cd sol && npm run build && pm2 restart plofolia-sol
    terra が起動失敗 --cwd なしで起動 pm2 delete plofolia-terra && pm2 start ... --cwd /home/ubuntu/plofolia/terra
    環境変数が反映されない --update-env なし pm2 restart plofolia-sol --update-env
    Nginx proxy先ポートが違う nginx設定ミス sudo cat /etc/nginx/sites-enabled/plotia でポート確認

    Step 4: Nginxエラーログ確認

    sudo tail -50 /var/log/nginx/error.log

    3. デプロイが失敗した¶

    titan/deploy.ps1 が途中で止まった¶

    # ローカルで確認
.\titan\deploy.ps1

    よくある失敗パターン:

    公開鍵プッシュのタイムアウト（60秒制限） → キーを再プッシュして再実行。スクリプトは冪等なので再実行可能。

    git push が rejected

    git pull origin develop --rebase
git push origin develop

    EC2側のビルドエラー（npm run build 失敗）

    # EC2にSSH接続してログを確認
ssh -i /e/DEV/plofolia/ec2_key ubuntu@13.115.254.4
cd ~/plofolia
pm2 logs plofolia-terra --lines 100 --nostream

    TypeScriptコンパイルエラーの場合はコードを修正してpush → 再デプロイ。

    terra ビルドがメモリ不足でクラッシュ（t3.micro）

    # EC2上でswapを確認（通常2GBのswapfileが存在する）
free -h
swapon --show

# PM2サービスを停止してメモリを確保してからビルド
pm2 stop all
cd ~/plofolia/terra
NODE_OPTIONS='--max-old-space-size=750' npm run build
pm2 start all
pm2 save

    ポイント: サービスを停止してから NODE_OPTIONS=--max-old-space-size=750 を付けてビルドすること。 サービス起動中のビルドはメモリ競合でOOMになることがある。

    DBマイグレーションを適用するのを忘れた¶

    # EC2上で最新マイグレーションを確認・適用
ssh -i /e/DEV/plofolia/ec2_key ubuntu@13.115.254.4
PGPASSWORD=polotiaakplt psql -h localhost -U plofolia -d plofolia \
  -f ~/plofolia/sol/src/db/migrations/<ファイル名>.sql
pm2 restart plofolia-sol

    4. DB接続エラー¶

    ログに ECONNREFUSED / password authentication failed / relation does not exist が出る場合。

    確認手順¶

    Step 1: PostgreSQLが起動しているか

    sudo systemctl status postgresql
sudo systemctl start postgresql  # 停止していれば起動

    Step 2: 接続確認

    PGPASSWORD=polotiaakplt psql -h localhost -U plofolia -d plofolia -c "SELECT 1;"

    Step 3: パスワード不一致

    sudo -u postgres psql -c "ALTER USER plofolia WITH PASSWORD 'polotiaakplt';"

    Step 4: テーブルが存在しない（マイグレーション未適用）

    PGPASSWORD=polotiaakplt psql -h localhost -U plofolia -d plofolia -c "\dt"

# 全マイグレーションを一括適用
for f in ~/plofolia/sol/src/db/migrations/*.sql; do
  echo "Applying $f..."
  PGPASSWORD=polotiaakplt psql -h localhost -U plofolia -d plofolia -f "$f"
done

    5. React hydration エラー（#418/#425/#423）¶

    ブラウザコンソールに Hydration failed / Text content does not match が出る、またはページが白くなる。

    原因¶

    Zustand の persist が localStorage から auth状態を復元するタイミングと、サーバーサイドレンダリングのタイミングがずれる。

    確認・対処¶

    発生しているコンポーネントを特定

    ブラウザDevTools → Console → エラーのスタックトレースからコンポーネント名を確認。

    修正パターン（実装済み）

    // authStore.ts の _hasHydrated フラグを使う
const { isAuthenticated, _hasHydrated } = useAuthStore()

// NG: SSR と CSR で値が異なり hydration mismatch になる
{isAuthenticated() && <AuthOnlyUI />}

// OK: hydration 完了後のみ条件付きレンダリング
{_hasHydrated && isAuthenticated() && <AuthOnlyUI />}

    または mounted パターン（Headerなど）:

    const [mounted, setMounted] = useState(false)
useEffect(() => setMounted(true), [])
const authed = mounted && isAuthenticated()

    修正済みコンポーネント: Header, TimelinePage, PostCard, PostDetail, TimedComments, ProfilePage, Sidebar, BottomNav

    6. 翻訳キー欠落クラッシュ（MISSING_MESSAGE）¶

    ページが白くなり、PM2ログに MISSING_MESSAGE: xxx.yyy (ja) が出る。

    確認手順¶

    Step 1: ログで欠落キーを特定

    pm2 logs plofolia-terra --lines 50 --nostream | grep MISSING

    Step 2: 該当キーを全5言語ファイルに追加

    翻訳ファイルは terra/messages/ 以下:

    • ja.json / en.json / zh.json / ko.json / es.json

    重要: 1言語でもキーが欠けていると、その言語でクラッシュする。必ず5ファイル全て更新すること。

    Step 3: 修正後にデプロイ

    .\titan\deploy.ps1

    7. ログイン・認証の不具合¶

    「ログインできない」¶

    1. メール認証が完了しているか確認（未認証ユーザーはログイン不可）
    2. パスワードが正しいか確認
    3. Aurora で状態確認:
    4. https://aurora.plofolia.com → ユーザー一覧 → 「メール未認証」フィルタ
    5. 対象ユーザーの詳細ページ → 「メール認証を手動で完了させる」ボタンを押す

    緊急時（DB 直接操作）:

    -- 状態確認
SELECT username, email, email_verified_at, email_verification_token
FROM users WHERE email = 'xxx@example.com';

-- 手動認証（Aurora から操作できない場合のみ）
UPDATE users SET email_verified_at = NOW(), email_verification_token = NULL
WHERE email = 'xxx@example.com';

    JWTが期限切れで弾かれる¶

    • アクセストークン: 15分で期限切れ → リフレッシュトークン（7日）で自動更新
    • リフレッシュも切れた場合は再ログインが必要（仕様通り）

    OAuth（Google/Twitter）ログインが失敗する¶

    • Googleコンソールのコールバック URL に https://plofolia.com/api/v1/oauth/google/callback が登録されているか確認
    • Twitterコンソールも同様に確認

    8. ファイルアップロードエラー¶

    400 Bad Request が返る¶

    • 動画MIMEタイプに codecs パラメータが含まれていないか確認（video/webm; codecs=vp9 → video/webm に変換済み）
    • ファイルサイズ制限（画像4枚、動画1本のみ、音声+カバー画像1枚）を超えていないか

    アップロードしたファイルが表示されない¶

    # EC2上でストレージパスを確認
ls -la ~/plofolia/sol/uploads/

    STORAGE_PROVIDER=local の場合、ファイルは EC2 の sol/uploads/ に保存される。

    9. SSE（リアルタイム通知/DM）が届かない¶

    確認手順¶

    1. ブラウザDevToolsの Network タブ → stream?token= で始まるリクエストが pending になっているか確認
    2. 25秒ごとに keepalive（: keepalive）が流れているか確認
    3. SSE接続が失敗している場合、フロントは60秒ポーリングにフォールバックする（通知は遅れるが届く）

    Nginx の SSE 設定確認¶

    Nginx のタイムアウトが短いと SSE接続が切れる。設定確認:

    sudo grep -A5 'stream' /etc/nginx/sites-enabled/plotia

    必要なディレクティブ:

    proxy_buffering off;
proxy_cache off;
proxy_read_timeout 3600s;

    10. 管理画面（Aurora）に入れない¶

    Aurora へのアクセス（SSHトンネル不要）¶

    https://aurora.plofolia.com で直接アクセス可能。

    1. HTTP Basic Auth ダイアログ:
    2. ユーザー名: aurora / パスワード: CLAUDE.local.md 参照
    3. Aurora ログイン画面:
    4. ユーザー名: admin / パスワード: Password123!
    5. MFA 設定済みならアプリの6桁コードを入力

    JWTは 15分で失効。操作中に認証エラーが出たら再ログインする。

    「HTTP Basic Auth ダイアログが繰り返し出る」¶

    Nginx レベルの認証失敗。CLAUDE.local.md に記載のパスワードを正確に入力する。
    パスワードを変更した場合は EC2 上で htpasswd ファイルを再生成:

    sudo htpasswd -bc /etc/nginx/.htpasswd_aurora aurora '新しいパスワード'
sudo systemctl reload nginx

    「Invalid credentials」でログインできない¶

    admin アカウントが存在するか確認:

    ssh -i /e/DEV/plofolia/ec2_key ubuntu@13.115.254.4
PGPASSWORD=polotiaakplt psql -h localhost -U plofolia -d plofolia \
  -c "SELECT username, role, email_verified_at FROM users WHERE role = 'admin';"

    admin が存在しない場合は seed を実行（本番DBへの seed は要注意）。

    メンテナンスモード¶

    Nginxレベルでメンテナンスページ（静的HTML）を有効にする方法。

    有効化（計画メンテナンス時）¶

    # メンテナンスモードON（全リクエストを503ページにリダイレクト）
sudo touch /var/www/plofolia-maintenance/enabled
sudo systemctl reload nginx

    解除¶

    # メンテナンスモードOFF
sudo rm /var/www/plofolia-maintenance/enabled
sudo systemctl reload nginx

    自動フォールバック（Next.js / API が落ちた場合）¶

    設定変更不要。Nginxが502/503/504エラーを検知すると /var/www/plofolia-maintenance/maintenance.html を自動で返す。

    メンテナンスページのHTML更新¶

    # ローカルで titan/maintenance.html を編集後、EC2にアップロード
scp -i E:/DEV/plofolia/ec2_key titan/maintenance.html ubuntu@13.115.254.4:/tmp/
ssh -i E:/DEV/plofolia/ec2_key ubuntu@13.115.254.4 \
  "sudo cp /tmp/maintenance.html /var/www/plofolia-maintenance/maintenance.html"

    緊急時コマンド一覧¶

    # 全サービスを再起動
pm2 restart all

# 全サービスのログをリアルタイムで確認
pm2 logs

# 特定サービスのログ（直近100行）
pm2 logs plofolia-sol --lines 100 --nostream

# PM2プロセス一覧
pm2 status

# Nginx設定テスト → リロード
sudo nginx -t && sudo systemctl reload nginx

# ディスク使用量確認
df -h

# メモリ使用量確認
free -h

# PostgreSQL接続確認
PGPASSWORD=polotiaakplt psql -h localhost -U plofolia -d plofolia -c "SELECT COUNT(*) FROM users;"
