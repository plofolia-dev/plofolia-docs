    フロントエンド¶

    Solarized Dark スタイル規則¶

    VideoPlayer・AudioPlayer・PostCard・タイムコメント等、全コンポーネントで Solarized Dark を統一する。

    背景色

    クラス 用途
    bg-sol-base03 最も暗い背景（メインコンテナ）
    bg-sol-base02 強調背景（選択状態・ホバー）
    bg-sol-base01 ボーダー・区切り線

    禁止事項

    • bg-black / bg-gray-* — 使用禁止。必ず bg-sol-base03 を使う
    • text-white / text-gray-* — 使用禁止。以下の Solarized テキストカラーを使う

    テキストカラー（明度順）

    クラス 用途
    text-sol-base3 強調テキスト（見出し・重要ラベル）
    text-sol-base2 通常テキスト（ユーザー名・本文）
    text-sol-base1 セカンダリテキスト
    text-sol-base0 補助テキスト
    text-sol-base01 プレースホルダー・ラベル
    text-sol-base03 濃い背景上のボタン文字（= #002b36）

    アクセントカラー

    text-sol-cyan（主アクセント）/ text-sol-blue / text-sol-yellow / text-sol-orange / text-sol-red / text-sol-green / text-sol-violet

    React Hydration エラー対応¶

    Next.js App Router で SSR と CSR の出力が食い違うと hydration エラー（#418/#425/#423）が発生する。 localStorage や Zustand persist を参照するコンポーネントで頻発する。

    パターン（authStore の _hasHydrated フラグ）

    // NG: hydration 前は undefined になりサーバーと不一致
if (isAuthenticated()) { return <UI /> }

// OK: hydration 完了フラグが立つまでレンダリングをスキップ
if (_hasHydrated && isAuthenticated()) { return <UI /> }

    _hasHydrated は useAuthStore の onRehydrateStorage コールバックで true にセットされる。 PostCard・Header・TimelinePage 等、auth に依存するコンポーネントで必ず使う。

    ブラウザキャッシュによる誤発生

    古い JS キャッシュが残っている場合も同じエラーが出る。 Ctrl+Shift+R または DevTools → Application → "Clear site data" で解消。

    メディアエンコード（terra/src/lib/mediaEncode.ts）¶

    ブラウザ内で録画・トリミング・再エンコードを行う。罠が多い。

    AudioContext の生成タイミング¶

    // NG: await より後に生成すると suspended 状態になり音声録音が失敗する
const result = await someAsyncOperation()
const ctx = new AudioContext() // suspended!

// OK: ユーザージェスチャーの同期コンテキストで先に生成
const ctx = new AudioContext() // resumed
const result = await someAsyncOperation()

    ブラウザのオートプレイポリシーで AudioContext はユーザー操作の同期コンテキスト外で生成すると suspended になる。音声が無音になるがエラーは出ないため気づきにくい。

    File.type に codecs を含めない¶

    // NG: multer が MIME パースに失敗して text/plain と判断 → 400 エラー
new File([blob], 'audio.webm', { type: 'audio/webm; codecs=opus' })

// OK
new File([blob], 'audio.webm', { type: 'audio/webm' })

    フォーマット選択¶

    メディア 出力形式 フォールバック
    動画 WebM（VP9/VP8） Safari は MP4
    音声 WebM/Opus 128kbps Opus 非対応ブラウザは WAV
    画像 WebP JPEG

    音声を WAV から WebM/Opus に変更した理由: WAV は無圧縮で 5分 ≈ 50MB。WebM/Opus なら ≈ 5MB。

    muted 動画の音声取得¶

    video.muted = true の状態では AudioContext.createMediaElementSource() が無音を返す。 回避策: 元ファイルを decodeAudioData() でデコード → AudioBufferSourceNode → MediaStreamDestination 経由でミックスする。

    NSFW 自動判定（terra/src/lib/nsfwDetect.ts）¶

    nsfwjs（MobileNetV2）をブラウザ内で実行して画像・動画の NSFW 確率を推定する。

    閾値

    hentai + porn > 0.6          → autoFlag（NSFW フラグを自動セット）
hentai + porn + sexy * 0.5 > 0.35  → suggest（フラグ推奨の警告表示）

    webpack 設定（terra/next.config.mjs）

    nsfwjs の全モデルシャードが Browserify 形式（require(factory, [])）の UMD で、webpack 5 が静的解析できずビルドエラーになる。以下の設定で回避している:

    // inception_v3 / mobilenet_v2_mid → スタブに置換（使わないので実行時も不要）
new wp.NormalModuleReplacementPlugin(
  /nsfwjs[\\/]dist[\\/]models[\\/](inception_v3|mobilenet_v2_mid)[\\/]/,
  stubPath,
)

// mobilenet_v2 → noParse（解析スキップ、実行時に UMD として正常動作）
config.module.noParse = /nsfwjs[\\/]dist[\\/]models[\\/]mobilenet_v2[\\/]/

    モデルのキャッシュ

    getModel() はモジュールレベルの model 変数でシングルトンを保持する。 二重ロードを防ぐため modelPromise でインフライトのPromiseも追跡する。

    タイムコメント（TimedComments.tsx）¶

    固定高さ必須¶

    // NG: コメントが増えるとコンテナが無限に縦伸びする
<div className="h-auto overflow-y-auto">

// OK: 固定高さを指定する
<div className="h-72 flex flex-col overflow-hidden">

    chatPanel prop のパス¶

    • 動画あり: VideoPlayer の chatPanel prop に <TimedComments /> → オーバーレイ表示
    • 動画なし・音声あり: AudioPlayer の chatPanel prop → プレイヤー下部に表示
    • 動画と音声に同時に chatPanel を渡さない（hasVideoMedia フラグで制御）

    PostForm ドラフト自動保存¶

    const DRAFT_KEY = 'plofolia_draft_post'
    • 入力から 2 秒後に localStorage に保存（デバウンス）
    • フォームマウント時に保存済みドラフトがあればバナーで通知
    • title / content / tags / isNsfw / postType を保存

    バックエンド¶

    SSE（Server-Sent Events）アーキテクチャ¶

    通知・DM 未読バッジのリアルタイム配信に SSE を使用。

    GET /notifications/stream?token=XXX
GET /messages/stream?token=XXX

    EventSource はカスタムヘッダーを送れないため JWT をクエリパラメータで渡す。 authenticate ミドルウェアが req.query.token にフォールバック対応済み。

    サーバー側管理

    const sseClients: Map<userId, Set<Response>> = new Map()

    一つのユーザーが複数タブを開いても対応できるよう Set で管理する。

    keepalive

    25 秒ごとにコメント行（: keepalive\n\n）を送信してコネクションを維持する。

    フロントエンド

    SSE 接続中はポーリングを抑制し、切断時は 60 秒フォールバックポーリングに切り替える。

    認証システム詳細¶

    JWT ペイロード

    {
  userId, username, email,
  isAdult,       // NSFW コンテンツ表示の可否に使用
  preferredLocale,  // next-intl の言語切替に使用
  role,          // user | admin | moderator
}

    ブラウザ言語検出と JWT 連動

    1. 未ログイン: next-intl middleware が Accept-Language ヘッダーで自動検出
    2. ログイン時: auth.service.ts の issueTokens() で users.preferredLocale を JWT に焼き込む
    3. フロント authStore.ts の setTokens() で JWT をデコードして NEXT_LOCALE cookie をセット
    4. 次ページ遷移から next-intl がその言語で表示

    ログインブロック条件

    emailVerificationToken IS NOT NULL AND emailVerifiedAt IS NULL

    つまり「メール送信済み・未確認」状態のみブロック。OAuth ユーザーはこのカラムが null なので影響なし。

    MFA デバイス信頼（Aurora 管理画面）

    信頼済みデバイスの仕組み:
  生成: crypto.randomBytes(32).toString('hex')  → 64 文字 hex
  保存: SHA-256 ハッシュをDBに保存（raw token は localStorage にのみ存在）
  検証: リクエストの raw token を SHA-256 → DB の hash と比較
  有効期限: 30日

    信頼済みデバイスから再ログインすると TOTP をスキップできる。

    モデレータ権限マトリクス

    操作 admin moderator
    ダッシュボード・閲覧 o o
    投稿削除・報告/バグ/問い合わせ対応 o o
    BAN / unban / ユーザー削除 o x
    監査ログ・招待コード o x
    アカウント管理（/accounts） o x
    • adminAuthenticate: admin + moderator が通過
    • requireAdmin: admin のみ通過

    最後の admin は降格不可: PATCH /admin/accounts/:userId/role で自分自身・最後の admin を変更しようとすると LAST_ADMIN エラーを返す。

    メディアアップロードパイプライン¶

    magic bytes 検証（二重チェック）

    1. multer の fileFilter で file.mimetype を確認（MIME 宣言チェック）
    2. magic-bytes.js でファイル先頭バイトを読んで実際のフォーマットを確認（MIME 偽装対策）

    両方通過したファイルだけが保存される。

    ファイル名

    uploads/<postId>/<uuid>.<ext>

    postId ディレクトリで投稿ごとに分離。削除時は postId ディレクトリごと消せる。

    OAuth（Twitter）実装¶

    passport-twitter を廃止して passport-oauth1 で直接実装（TwitterOAuth1Strategy）。

    廃止理由: passport-twitter が xmldom 経由で critical CVE（xtraverse）を持っていた。

    Twitter OAuth 1.0a はリクエストトークンをセッションで管理する必要があるため session: true を指定する。 OAUTH_CALLBACK_BASE を http://localhost:3000（Next.js）にするとセッション Cookie が失われて認証が通らない。必ず http://localhost:4000（sol 直）にすること。

    データモデル¶

    DB 名の経緯¶

    DB名・ユーザー名は旧称 plotia だったが 2026-04-30 にリネーム済み。

    ローカル Docker と EC2 本番でクレデンシャルが異なる:

    環境 DB名 ユーザー パスワード
    ローカル Docker plofolia plofolia plofolia_secret
    EC2 本番 plofolia plofolia （CLAUDE.local.md 参照）

    クリエイタータイプ マルチ選択¶

    users.creatorTypeId  ← 後方互換のため主タイプを保持し続ける
user_creator_types   ← マルチ選択の実体（userId, creatorTypeId, isPrimary）

    API では creatorTypeCodes: string[] で送る（最大 5 つ、先頭が主タイプ）。 viewer は他タイプと排他（選ぶと他がクリア、他を選ぶと viewer がクリア）。

    timedCommentCount の取得¶

    タイムラインと個別投稿の両方で timedCommentCount が必要なため、2 か所に同じサブクエリを追加している。

    // sol/src/api/posts/posts.service.ts の getPostById 等
timedCommentCount: sql<number>`(SELECT COUNT(*)::int FROM timed_comments WHERE post_id = ${posts.id})`

// sol/src/api/timeline/timeline.service.ts の buildPostSelect() にも同じものが必要
// ここを忘れると timeline/global や timeline/following でだけ count が 0 になる

    同一メールで複数アカウント可¶

    users.email の unique 制約を削除済み（migration 0024）。ユーザー名のみが一意制約を持つ。

    ログイン identifier 方式: @ を含む文字列はメールアドレス、含まない場合はユーザー名として DB 検索する。

    PostCard の NSFW 表示状態¶

    状態 動作
    未ログイン 投稿は表示。メディアを「R18 — ログインして閲覧」リンクに差し替え（クリック解除不可）
    ログイン済み・成人・hideNsfw=false メディアを表示（blurNsfw=true の場合はクリックで blur 解除）
    ログイン済み・成人・hideNsfw=true return null（完全非表示）
    ログイン済み・非成人 return null（完全非表示）

    デフォルト: hideNsfw: true / blurNsfw: true（新規ユーザーは NSFW 初期非表示）。

    多言語対応¶

    翻訳ファイルは 5 言語すべてを同時に更新すること。

    terra/messages/ja.json
terra/messages/en.json
terra/messages/zh.json
terra/messages/ko.json
terra/messages/es.json

    1 ファイルでも欠けると MISSING_MESSAGE エラーでページ全体がクラッシュする（白画面）。

    マイグレーション¶

    ローカル

    cd sol
npm run db:generate   # schema/*.ts の変更からSQLを生成
npm run db:migrate    # ローカル Docker に適用

    EC2 本番（drizzle-kit push は使わず SQL 直接適用）

    PGPASSWORD=<pw> psql -h localhost -U plofolia -d plofolia \
  -f ~/plofolia/sol/src/db/migrations/0033_xxxx.sql

    drizzle.__drizzle_migrations テーブルに適用履歴が記録される。 マイグレーション番号が飛んでいる場合は全件確認して欠けている番号を適用する。

    既知の罠・トラブルシューティング¶

    現象 原因と対処
    terra の hot reload が効かなくなった npm run build をローカルで実行してしまった。.next/ を削除して next dev を再起動
    Twitter OAuth がローカルで失敗する OAUTH_CALLBACK_BASE=http://localhost:3000 になっている。http://localhost:4000 に変更
    ソーシャルログインが動かない Google/Twitter コンソールにコールバック URL を登録していない。LOCAL_DEV.md の OAuth セクションを参照
    タイムラインだけ timedCommentCount が 0 timeline.service.ts の buildPostSelect() にサブクエリを追加し忘れている
    動画投稿で音声が録音されない AudioContext を await より後で生成している。ユーザージェスチャーの同期コンテキストで先に生成すること
    動画アップロードが 400 になる File.type に video/webm; codecs=vp9 など codecs が入っている。video/webm のみにする
    翻訳追加後に白画面 5 言語すべてのメッセージファイルに追加していない
    React hydration error #418 _hasHydrated を確認していないコンポーネントが auth 状態を直接参照している
    pm2 で環境変数が反映されない NEXT_PUBLIC_* 変数はビルド時に焼き込まれるため npm run build してから pm2 restart --update-env
    aurora にアクセスできない https://aurora.plofolia.com は HTTP Basic Auth あり。認証情報は CLAUDE.local.md を参照
