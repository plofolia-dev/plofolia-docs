    ローカル開発環境セットアップ¶

    Windows PC + Docker Desktop での開発環境構築手順。

    前提条件¶

    • Node.js 20+
    • Docker Desktop（WSL2モードで動作）
    • Git

    サービス構成¶

    フォルダ 役割 ポート
    sol/ メインバックエンド（Express API） 4000
    terra/ メインフロントエンド（Next.js） 3000
    aurora/ 管理フロントエンド（Next.js） 3001
    nova/ 管理バックエンドランナー 4001

    開発フロー¶

    ローカルで動作確認 → コミット → develop に push → EC2 デプロイ

    DB スキーマを変更したときは必ずマイグレーションファイルを生成してコミットする。
    EC2 では drizzle-kit push ではなく SQL ファイルを直接適用する。

    初回セットアップ¶

    1. リポジトリ確認・ブランチ切替¶

    cd E:\DEV\plofolia
git checkout develop

    2. Docker で DB 起動¶

    npm run db:up
# または
docker compose up -d

    確認：

    docker compose ps
# plofolia_db が healthy になっていればOK

    pgAdmin: http://localhost:5050
    Email: admin@plofolia.dev / Password: admin

    3. sol（メインバックエンド）セットアップ¶

    cd E:\DEV\plofolia\sol
npm install

    .env は既存のファイルをそのまま使用（plofolia:plofolia_secret@localhost:5432/plofolia）。
    .env.example からコピーして編集する場合は以下を参考に：

    NODE_ENV=development
PORT=4000
FRONTEND_URL=http://localhost:3000

DATABASE_URL=postgresql://plofolia:plofolia_secret@localhost:5432/plofolia

JWT_SECRET=localsecret1234567890localsecret1234567890localsecret1234567890ab
JWT_ACCESS_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d

STORAGE_PROVIDER=local
UPLOAD_DIR=./uploads

ADMIN_JWT_SECRET=163abc8203d365d6587f1f3ed3430cbbe85f23bcac3e7442e559c9b85ef8b006ebe8b89225cfb92393a2def8b4bb9f6b
ADMIN_PORT=4001
ADMIN_FRONTEND_URL=http://localhost:3001

# OAuth（ローカル開発用）
OAUTH_CALLBACK_BASE=http://localhost:4000

    ソーシャルログインを動作させるには（コンソール登録が必要）¶

    重要: OAuth は Google/Twitter コンソールにコールバック URL を登録しないと動作しない。

    Google Cloud Console:

    1. https://console.cloud.google.com/apis/credentials
    2. 対象の OAuth クライアント → 「承認済みのリダイレクト URI」に追加:
    3. http://localhost:4000/api/v1/oauth/google/callback
    4. https://plofolia.com/api/v1/oauth/google/callback（本番用）

    Twitter Developer Portal:

    1. https://developer.twitter.com/en/portal/projects
    2. 対象の App → Settings → Authentication settings → Callback URL に追加:
    3. http://localhost:4000/api/v1/oauth/twitter/callback
    4. https://plofolia.com/api/v1/oauth/twitter/callback（本番用）

    DB スキーマとシードを適用：

    npm run db:migrate   # マイグレーション適用
npm run db:seed      # 基本テストデータ投入（8ユーザー + 19投稿）

    初回は全マイグレーションを一括適用するコマンドが便利：

    # Dockerコンテナ経由でSQLを全て適用
for f in src/db/migrations/*.sql; do
  docker exec -i plofolia_db psql -U plofolia -d plofolia < "$f"
done
npm run db:seed

    シードデータの種類（フル投入する場合）¶

    コマンド 内容 所要時間
    npm run db:seed 基本8ユーザー・19投稿・フォロー・いいね 数秒
    npm run db:seed:media 動画/音声/画像の多様性テスト用投稿 + タイムコメント66件 数秒
    npm run db:seed:assets 外部からメディアファイルをDLしてエンコード（ffmpeg必須・初回のみ） 数分
    npm run db:seed:bulk 合計1000件になるまで大量投稿を生成（50名追加） 数十秒
    npm run db:seed:viral バズ投稿5件 + タイムコメント750件 数秒
    npm run db:seed:influencers インフルエンサー3名 + 超バズ投稿 + 大量タイムコメント約9920件 数秒
    npm run db:seed:reset -- --confirm 全データ削除（NODE_ENV=production では実行不可） 即時

    実行順: seed → seed:media → seed:assets → seed:bulk → seed:viral → seed:influencers

    注意: seed:assets は ffmpeg が必要。未インストールの場合は音声が WebM 変換されず外部URL（SoundHelix）が DB に残る。
    ffmpeg インストール: winget install ffmpeg（Windows）または sudo apt install ffmpeg（EC2）

    動作確認: http://localhost:4000/health → {"status":"ok"}

    4. terra（メインフロントエンド）セットアップ¶

    cd E:\DEV\plofolia\terra
npm install

    .env.local を作成（cp .env.example .env.local または直接作成）：

    NEXT_PUBLIC_API_URL=http://localhost:4000/api/v1
NEXT_PUBLIC_APP_NAME=Plofolia

    注意: terra では npm run build を実行してはいけない。 .next/ が production artifact で上書きされ、next dev が壊れる。 ローカルは常に next dev（hot reload）で動かすこと。ビルドが必要な検証は EC2 で行う。

    5. aurora / nova（管理画面）セットアップ¶

    cd E:\DEV\plofolia\nova && npm install
cd E:\DEV\plofolia\aurora && npm install

    aurora の .env.local：

    ADMIN_API_URL=http://localhost:4001

    毎回の起動手順（2回目以降）¶

    cd E:\DEV\plofolia

# 1. DB 起動（Docker が停止していた場合）
npm run db:up

# 2. 全サービス一括起動（sol + terra + nova + aurora）
npm run dev
    URL 用途
    http://localhost:3000/ja/timeline メインアプリ（terra）
    http://localhost:4000/health sol API ヘルスチェック
    http://localhost:3001 管理画面（aurora）
    http://localhost:4001/health nova 管理API ヘルスチェック

    部分起動¶

    npm run dev:app    # アプリのみ（sol + terra + Cloudflare Tunnel）
npm run dev:admin  # 管理画面のみ（nova + aurora）

    各サービスは tsx watch（sol/nova）・Next.js HMR（terra/aurora）により自動リロードされます。

    Cloudflare Tunnel（スマホ・別PCから確認）¶

    dev:app には Cloudflare Tunnel が組み込まれており、起動時にターミナルに以下のような URL が表示されます。

    [TUNNEL] | https://xxxx-xxxx.trycloudflare.com |

    この URL でスマホや別 PC からローカル環境（terra）に直接アクセスできます。

    • URL は起動のたびにランダムで変わります
    • アカウントなし・設定なしで即使用可能（trycloudflare.com の無料枠）
    • aurora（管理画面・port 3001）も公開したい場合は npm run tunnel を使う

    DB スキーマを変更したとき¶

    cd E:\DEV\plofolia\sol

# 1. schema/*.ts を編集
# 2. マイグレーションファイルを生成
npm run db:generate

# 3. ローカルに適用して動作確認
npm run db:migrate

# 4. コミット（migrations/ ファイルを含める）
git add src/db/schema/ src/db/migrations/
git commit -m "feat: ..."

# 5. git push → CI 通過後に EC2 デプロイ

    EC2 での適用は ../ops/DEPLOY.md を参照。

    管理画面へのアクセス¶

    1. npm run dev（または npm run dev:admin）を実行
    2. http://localhost:3001 にアクセス
    3. メール admin@example.com / パスワード Password123! でログイン

    管理者JWTは 8時間で失効します。

    スマホ・他PCから確認する場合¶

    npm run dev:app を使えば terra の Cloudflare Tunnel が自動で起動します（別途コマンド不要）。

    aurora（管理画面）も同時に公開したい場合は別ターミナルで：

    cd E:\DEV\plofolia
npm run tunnel

    バグ報告の確認・対応¶

    ユーザーからの報告を確認する（Claude Code で参照）¶

    # open バグ一覧
npm run bugs

# 全ステータス
npm run bugs all

# 特定バグの詳細
node titan/bugs.js <UUID>

    スクリプトはローカル Docker の DB に直接接続して出力します。

    aurora 管理画面での対応¶

    http://localhost:3001 → 「バグ報告」メニュー → ステータス変更・管理者メモを記入

    GitHub Issues と Claude Code の連携¶

    タスク・バグ・機能要望は GitHub Issues で管理する。Claude Code は GitHub MCP サーバーを通じて Issues を直接参照・作成できる。

    Claude Code に MCP を設定する¶

    .mcp.json（プロジェクトルート・gitignored）に以下を記載し、Claude Code を再起動する:

    {
  "mcpServers": {
    "github": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxxxxxx"
      }
    }
  }
}

    または Claude Code CLI で:

    claude mcp add github -s project \
  -e "GITHUB_PERSONAL_ACCESS_TOKEN=ghp_xxxxxxxx" \
  -- npx -y @modelcontextprotocol/server-github

    Issues を参照する¶

    Claude Code に GitHub Issues を確認して と伝えると、現在のオープンIssue一覧を取得できる。

    • Issueリスト: https://github.com/plofolia-dev/plofolia/issues
    • ラベル・優先度の詳細は CONTRIBUTING.md を参照

    停止手順¶

    # 各サービス: Ctrl+C

# DB 停止
npm run db:down
